# **Phase 0: Foundation Building (Pre-University/Review)**
*Duration: 1-3 months depending on background*

## **Module 0.1: Diagnostic & Mindset Setup**
*Duration: 1 week*

### **Self-Assessment**
- **Prerequisite Knowledge Check**:
  - Can you solve: 3(4 + 5 ÷ 2) - 7²?
  - Can you factor: x² - 5x + 6?
  - Can you solve: 2x + 3y = 7 and x - y = 1 simultaneously?
  - Can you calculate: 15% of 240?
  - Can you convert: 3/8 to decimal and percentage?

- **Mathematical Maturity Assessment**:
  - Comfort with abstract symbols
  - Persistence in problem-solving
  - Ability to follow multi-step procedures
  - Comfort with being wrong and learning from mistakes

### **Learning Infrastructure Setup**
- **Tools**:
  - Python/Jupyter Notebook for computational practice
  - Desmos/GeoGebra for visualization
  - Anki/Notion for spaced repetition and note-taking
  - Quality notebook and graphing paper

- **Study System**:
  - Daily practice (30-60 minutes) > weekly cramming
  - "Feynman Technique": Teach concepts back
  - Error journal: Document and analyze mistakes
  - Project-based learning approach

## **Module 0.2: Arithmetic & Number Systems**
*Duration: 2-3 weeks*

### **Core Topics with CS Connections**:

#### **1. Number Systems & Representation**
- **Integers**: Operations, properties (commutative, associative, distributive)
  - *CS Connection*: How computers represent integers (two's complement)
  - *Practice*: Implement addition using bit operations
- **Rational Numbers**: Fractions, decimals, percentages
  - *CS Connection*: Floating-point representation limitations
  - *Practice*: Write code to detect floating-point errors
- **Prime Numbers & Factorization**
  - *CS Connection*: Cryptography foundations
  - *Practice*: Implement Sieve of Eratosthenes algorithm

#### **2. Operations & Properties**
- **Order of Operations (PEMDAS/BODMAS)**
  - *Deep Dive*: Why multiplication before addition?
  - *CS Connection*: Parser design in compilers
  - *Practice*: Write a simple expression evaluator
- **Modular Arithmetic**
  - *CS Connection*: Hash functions, circular buffers
  - *Practice*: Implement a circular queue using modulo
- **Exponents & Radicals**
  - *CS Connection*: Algorithm complexity (O(2ⁿ) vs O(√n))
  - *Practice*: Compare exponential vs polynomial growth empirically

#### **3. Ratios, Proportions, Percentages**
- **Direct/Inverse Variation**
  - *CS Connection*: Scaling algorithms, load balancing
  - *Practice*: Model cache hit ratios
- **Percentage Calculations**
  - *CS Connection*: Probability, statistics, data normalization
  - *Practice*: Implement data normalization for machine learning

### **Problem Sets with Progressive Difficulty**:

**Basic**:
1. Simplify: 48 ÷ (4 × 2) + 3²
2. Convert 5/16 to decimal and percentage
3. Find GCD and LCM of 24 and 36

**Intermediate**:
1. Solve without calculator: (0.75 × 64) ÷ (1.5 × 8)
2. A number increased by 25% gives 75. Find the number.
3. If a ≡ 3 (mod 7) and b ≡ 5 (mod 7), find (a × b) mod 7

**Advanced/CS-Focused**:
1. Implement a function that converts decimal to binary
2. Write code to simulate integer overflow in two's complement
3. Create a visualization of prime number distribution
