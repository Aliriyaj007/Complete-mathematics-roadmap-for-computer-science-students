# **Phase 0: Foundation Building (Pre-University/Review)**
*Duration: 1-3 months depending on background*

## **Module 0.3: Elementary Algebra**
*Duration: 3-4 weeks*

### **Core Topics with CS Connections**:

#### **1. Linear Equations & Inequalities**
- **Solving Techniques**:
  - Isolating variables
  - Systems of equations (substitution, elimination)
  - *CS Connection*: Constraint satisfaction problems
  - *Practice*: Solve resource allocation problems

- **Graphing Fundamentals**:
  - Slope-intercept form: y = mx + b
  - *CS Connection*: Linear regression basics
  - *Practice*: Plot CPU usage over time

#### **2. Polynomials & Factoring**
- **Operations**:
  - Addition, subtraction, multiplication
  - *CS Connection*: Polynomial-time algorithms
  - *Practice*: Implement polynomial multiplication

- **Factoring Techniques**:
  - GCF, difference of squares, trinomials
  - *CS Connection*: Error-correcting codes (Reed-Solomon)
  - *Practice*: Factor expressions programmatically

#### **3. Functions & Relations**
- **Function Notation**: f(x), domain, range
- **Function Composition**: f(g(x))
  - *CS Connection*: Pipeline architecture, function composition in programming
  - *Practice*: Implement function composition in Python
- **Inverse Functions**
  - *CS Connection*: Encryption/decryption pairs
  - *Practice*: Create simple Caesar cipher with inverse

#### **4. Word Problems → Algorithm Design**
- **Translation Process**:
  1. Read and understand the problem
  2. Define variables
  3. Write equations
  4. Solve
  5. Interpret results
- *CS Connection*: Algorithm specification
- *Practice*: Convert word problems to pseudocode

### **Problem Sets with Progressive Difficulty**:

**Basic**:
1. Solve: 3(x - 4) = 2x + 5
2. Factor: x² + 7x + 12
3. Graph: y = 2x - 3

**Intermediate**:
1. Solve system: 2x + y = 10, x - 3y = -5
2. If f(x) = 3x - 2 and g(x) = x², find f(g(3))
3. A rectangle has length 3 more than width. Area is 40. Find dimensions.

**Advanced/CS-Focused**:
1. **Algorithm Design**: "You have an array of numbers. Find two numbers that sum to a target value." Write the algebraic formulation.
2. **Function Implementation**: Write a Python function that takes coefficients of a quadratic and returns roots
3. **Visualization**: Create an interactive plot showing how changing slope affects a line

## **Module 0.4: Mathematical Thinking & Problem-Solving**
*Duration: 2 weeks*

### **Critical Skills Development**:

#### **1. Problem Decomposition**
- Break complex problems into simpler parts
- *CS Connection*: Modular programming, function design
- *Practice*: Decompose "calculate final grade" into steps

#### **2. Pattern Recognition**
- Identify sequences, repetitions, structures
- *CS Connection*: Algorithm design, data structure selection
- *Practice*: Find patterns in number sequences, predict next values

#### **3. Logical Reasoning**
- If-then reasoning, contrapositives
- *CS Connection*: Conditional statements in programming
- *Practice*: Translate word problems to logical statements

#### **4. Estimation & Approximation**
- Order-of-magnitude reasoning
- *CS Connection*: Algorithm efficiency estimation
- *Practice*: Estimate time complexity before coding

### **CS-Specific Applications**:

#### **1. From Algebra to Algorithms**:
- **Example**: "A store sells apples for $0.50 and oranges for $0.75. You have $10 and want at least 5 apples. How many of each can you buy?"
  - Algebraic formulation
  - Brute-force solution
  - Efficient solution using inequalities
  - *Implementation*: Write both solutions, compare efficiency

#### **2. Mathematical Modeling Basics**:
- **Simple Models**:
  - Linear growth: y = mx + b (CPU load over users)
  - Exponential growth: y = a·bˣ (viral content spread)
  - *Practice*: Model cache hit rate as function of cache size

#### **3. Error Analysis**:
- **Types of Errors**:
  - Rounding errors (floating-point arithmetic)
  - Approximation errors (early algorithm termination)
  - *Practice*: Compare exact vs approximate solutions

## **Module 0.5: Assessment & Transition Planning**
*Duration: 1 week*

### **Comprehensive Assessment**:

#### **1. Conceptual Understanding** (Quiz):
1. Explain why (-a) × (-b) = ab
2. Demonstrate three different factoring techniques
3. Explain domain restrictions in functions

#### **2. Problem-Solving** (Test):
**Section A**: Computation
- Solve: \(\frac{3}{4} + \frac{2}{5} \times \frac{10}{3}\)
- Simplify: \(2^3 \times 2^4 ÷ 2^2\)

**Section B**: Algebra
- Solve: \(x^2 - 5x = 14\)
- Find equation of line through (2,3) and (4,7)

**Section C**: Word Problems
- "A car rental charges $25/day + $0.15/mile. You have $200. Write inequality for possible rentals."

**Section D**: CS Application
- Write pseudocode to solve quadratic equation
- Estimate operations needed to sort 1000 items using bubble sort

#### **3. Practical Implementation** (Project):
**Option 1**: Simple Calculator with History
- Supports order of operations
- Stores calculation history
- Handles basic errors

**Option 2**: Function Plotter
- Takes function as input
- Plots graph with adjustable domain
- Marks key points (roots, intercepts)

**Option 3**: Number Theory Toolkit
- Prime number checker
- GCD/LCM calculator
- Modular arithmetic operations

### **Transition to Phase 1 Readiness Check**:

#### **Minimum Competencies**:
- ✅ Solve multi-step arithmetic problems without calculator
- ✅ Manipulate algebraic expressions confidently
- ✅ Translate word problems to equations
- ✅ Understand function notation and graphing basics
- ✅ Write simple pseudocode for mathematical procedures

## **Common Pitfalls & Solutions**:

### **Pitfall 1: Calculator Dependency**
- **Solution**: Designated "no calculator" days
- **Practice**: Mental math drills

### **Pitfall 2: Procedural without Understanding**
- **Solution**: "Explain why" exercises
- **Practice**: Teach concepts to someone else

### **Pitfall 3: Isolated Learning**
- **Solution**: Study groups, online forums
- **Practice**: Pair programming on math problems

### **Pitfall 4: Anxiety with Symbols**
- **Solution**: Gradual exposure therapy
- **Practice**: Start with concrete numbers, abstract gradually

## **Success Indicators for Phase 0 Completion**:

You're ready for Phase 1 when you can:
1. Look at a multi-step problem and devise a solution strategy
2. Explain mathematical concepts in plain English
3. Implement simple mathematical algorithms in code
4. Recognize when a problem is beyond current skills and know how to bridge the gap
5. Feel curiosity rather than dread when encountering new notation
